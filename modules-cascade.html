cascade

https://github.com/transpect/cascade

This module is a library of various XSLT stylesheets and XProc steps to generate a cascade paths document and to load files dynamically by the provided paths document.

Source ⬇

tr:paths

This step takes a cascade configuration and a XSLT stylesheet as input. The cascade configuration specifies the possible configuration levels (e.g. global, imprint, series, work). The XSLT stylesheet is used to override certain functions in paths.xsl, which provides the paths parameter document. The paths document provides an ordered collection of file paths.

Import URI

<p:import href="http://transpect.io/cascade/xpl/paths.xpl"/>

Synopsis

<tr:paths xmlns:tr="http://transpect.io">
  <p:input port="conf"/>                                  <!-- Cascade configuration according to this schema -->
  <p:input port="stylesheet"/>                            <!-- XSLT to override default paths.xsl -->
  <p:output port="result"/>                               <!-- The paths parameter document -->
  <p:with-option name="debug" select="'yes'"/>            <!-- "yes"|"no" -->
  <p:with-option name="debug-dir-uri" select="'debug'"/>  <!-- "debug" -->
  <p:with-option name="status-dir-uri" select="'status'"/><!-- "status" -->
</tr:paths>

tr:load-cascaded

The XProc pipleline loads the most specific file based on a cascade paths document. A fallback can be provided, if there exist no file under the current cascade path.

Import URI

<p:import href="http://transpect.io/cascade/xpl/load-cascaded.xpl"/>

Synopsis

<tr:load-cascaded xmlns:tr="http://transpect.io">
  <p:input port="paths"/>                                 <!-- The paths parameter document  -->
  <p:output port="result"/>                               <!-- Provides the loaded file -->
  <p:with-option name="filename" select="'yes'"/>         <!-- file to be loaded -->
  <p:with-option name="fallback" select="'yes'"/>         <!-- provide a fallback path -->
  <p:with-option name="debug" select="'yes'"/>            <!-- "yes"|"no" -->
  <p:with-option name="debug-dir-uri" select="'debug'"/>  <!-- "debug" -->
  <p:with-option name="status-dir-uri" select="'status'"/><!-- "status" -->
</tr:load-cascaded>

tr:get-clades-from-dirs

Generates a cascade configuration from a recursive directory listing and a collection of parameter sets. This configuration can be used by tr:paths ➼ as input.

Import URI

<p:import href="http://transpect.io/cascade/xpl/get-clades-from-dirs.xpl"/>

Synopsis

<tr:get-clades-from-dirs xmlns:tr="http://transpect.io">
  <p:input port="params"/>                                <!-- The initial params document. It's base URI is used to iterate over the subdirectories and to generate the cascade configuration. -->
  <p:output port="directory-param-sets"/>                 <!-- Provides the cascade configuration, later to be used by tr:paths -->
  <p:with-option name="debug" select="'yes'"/>            <!-- "yes"|"no" -->
  <p:with-option name="debug-dir-uri" select="'debug'"/>  <!-- "debug" -->
  <p:with-option name="status-dir-uri" select="'status'"/><!-- "status" -->
</tr:get-clades-from-dirs>
